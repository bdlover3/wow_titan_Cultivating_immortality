## 模型动画修复（AnimationDataEnum）

### 问题根因
- `SetAnimation(anim, variation)` 的 anim 参数是 **AnimationDataEnum**（标准 .m2 动画类型索引），不是 Emote ID
- 旧代码使用了错误ID：EMOTE_DANCE=94, EMOTE_SHY=63, EMOTE_KISS=58 等都是错误的
- 正确的 AnimationDataEnum 值：EmoteTalk=60, EmoteDance=64, EmoteLaugh=65, EmoteWave=66 等

### 修复内容（SisterModel.lua）
- 重写 `UI.ANIM` 表，使用正确的 AnimationDataEnum 值
- 新增 `ModelHasAnim(mf, animId)` 函数：用 `HasAnimation()` API 检测模型是否支持某动画
- 新增 `TrySetAnimationFromList(mf, animList, variation)` 函数：从候选列表中选择第一个模型支持的动画
- 修改 `PlayTalkAnimation`：使用候选列表 `{60, 8, 1}` 自动选择可用的说话动画
- 修改 `PlayDanceAnimation/PlayLaughAnimation`：使用候选列表 + IDLE 兜底
- 修改 `StartIdleAnimation`：所有空闲动画使用候选列表系统
- 新增 `/wcsi probe` 命令：探测当前模型支持的动画（0-150范围扫描）
- 新增 `/wcsi animkit <ID>` 命令：测试 PlayAnimKit API
- 修改 `/wcsi anim <ID>` 命令：播放前检测 HasAnimation，不支持则提示

### 参考来源
- `Blizzard_APIDocumentationGenerated/FrameAPICharacterModelBaseDocumentation.lua` — API定义
- `Blizzard_SharedXML/Classic/ModelFrames.lua` — Model_ApplyUICamera 使用 FreezeAnimation/SetAnimation(0,0)
- 暴雪官方代码中 CharacterFrame/DressUpFrame/InspectUI 均不使用 SetAnimation 播放表情动画
